1. 프로젝트 개요 및 비즈니스 전략[AGENT INSTRUCTION]
- 이 플랫폼은 공공데이터, GIS 공간정보, 생성형 AI를 결합한 지능형 산업단지 입지 매칭 서비스입니다.  코딩 시 **"공간 연산 경량화", "동적 가중치 연동 안정성", "실시간 반응형 UI"**를 최우선 원칙으로 삼으세요.산재한 공공데이터 사일로(Data Silo)를 가치사슬 중심으로 통합하고, 공간통계학(GeoPandas)과 다기준 의사결정과학(TOPSIS)을 융합하여 부울경 특화 제조 기업의 실패 없는 행정·물류·투자 의사결정을 지원합니다.  
1.1 타겟 도메인 매트릭스 (5대 평가 축)
- 지역여건: 지자체 낙후도 지수, 오염물질 배출사업장수 기반 지역 여건 평가 (환경오염물질_배출사업장수_norm, 낙후도지수_norm)
- 물류여건: 고속도로, 철도화물역, 항만, 공항 접근성 및 거리 여건 평가 (공항거리_norm, 항만거리_norm, 고속도로거리_norm, 화물역거리_norm)
- 산업혁신여건: 대학교 개수, 연구소전담부서 개수 기반 산학연 혁신 여건 평가 (univ_count_norm, 연구소전담부서_norm)
- 생활정주여건: 마트, 편의점, 병원, 약국 개수 기반 정주 인프라 여건 평가 (mart_count_norm, store_count_norm, hospital_count_norm, pharmacy_count_norm)
- 근로자이동여건: 버스정류장, 지하철역 개수 기반 대중교통 및 이동성 여건 평가 (bus_count_norm, subway_count_norm)

2. 핵심 의사결정 알고리즘 및 분석 파이프라인 아키텍처[AGENT INSTRUCTION]
공간 분석 및 추천 알고리즘이 추후 확정되는 시점에 유연하게 연동 가능하도록 객체지향적인 `BaseSpatialPipeline` 추상 인터페이스 구조를 구현합니다.
사용자가 직접 로드한 후보지 및 인프라 실데이터에 대하여 LLM 가중치 및 수동 보정 가중치(총합 100점 고정)를 실시간으로 반영하여 정합성을 검증합니다.
2.1 파이프라인 확장 구조 및 데이터 규격통합 연산 인터페이스: `run_analysis(candidate_gdf, infrastructure_gdf, weights)`의 공통 규격을 엄격히 준수합니다.
입력 데이터 좌표계 검증: 공간 분석 정확도 보장을 위해 후보지와 인프라 레이어 간의 CRS(좌표계) 일치 여부를 실시간 검증하고 필요시 EPSG:4326(또는 타겟 로컬 좌표계)로 변환을 수행합니다.
2.2 지표별 독립 연산 모듈화: 각 지표별 점수는 단순 평균 대신 문서에 기재된 고정 비율을 바탕으로 가중합을 계산합니다.
- 지역여건: 환경오염물질 배출사업장수_norm (0.6) + 낙후도지수_norm (0.4)
- 물류여건: 항만거리_norm (0.4) + 고속도로거리_norm (0.35) + 화물역거리_norm (0.15) + 공항거리_norm (0.1)
- 산업혁신여건: 연구소전담부서_norm (0.6) + univ_count_norm (0.4)
- 생활정주여건: hospital_count_norm (0.35) + pharmacy_count_norm (0.25) + mart_count_norm (0.25) + store_count_norm (0.15)
- 근로자이동여건: bus_count_norm (0.8) + subway_count_norm (0.2)
2.3 Mock 배제 및 결측 처리: 가짜(Mock) 데이터를 원천 배제하고 단일 정규화 데이터(`산업단지_정규화데이터.csv`)에서 유효 세부 컬럼의 점수를 세부 가중치에 맞춰 반영합니다.

3. 공간 분석 및 UI 파이프라인 흐름 (Streamlit)[AGENT INSTRUCTION]
실제 분석 알고리즘과 관계없이, 데이터 적재 및 시각화 파이프라인의 안전성을 보장하기 위해 GeoPandas와 Streamlit, 그리고 Folium 지도를 유기적으로 연동합니다.
서비스 구동 시, Neon DB(PostgreSQL)에 임시 이식한 `dan_integrated` 통합 데이터를 최우선적으로 쿼리하여 메모리 상에서 실시간 MinMax 정규화를 수행하며(DB 연결 불가 시 로컬 CSV 파일 백업 로드 지원), 사이드바에서 Gemini LLM이 가중치를 추천하면 이와 연계된 슬라이더 및 숫자 입력 칸이 양방향으로 동적으로 동기화(각 지표별 가중치 독립 조절 가능 및 100점 초과 시 경고 노출)되며, 세부조정 위젯 바로 하단에서 분석 실행을 즉시 트리거하는 흐름을 제공합니다. (이때 사용자가 분석 실행 후 슬라이더를 추가 조작해도 점수 결과 표의 만점 배점이 이전 분석 시점의 결과와 일관성을 갖도록 실제 분석 완료 시점의 가중치 상태(analyzed_weights)를 따로 보관하여 매칭 정합성을 보장합니다.)
분석 결과는 좌측 Folium 지도에 상위 5개 후보지만 마킹되며(이때 VWorld API와 AI 에이전트를 통해 사용자가 입력한 사업 설명에 대해 자격 검증을 순차 수행하여 가능/조건부 가능 판정된 단지만 필터링하여 상위 5개 노출, 자격 뱃지 시각화, 좌표 우선순위: 1순위 DAN_COORD_MAP(카카오 API로 수집된 dan_coords.csv 기반, 부울경 291개 중 278개 WGS84 좌표 확보), 2순위 geometry 실좌표(location_df에 lat/lon 컬럼 존재 시), 3순위 location_df CSV lat/lon, 4순위 LLM 추론 좌표(한국 범위 33~39N·124~132E 유효성 검증 후 사용), 병원·편의점 반경 필터링도 동일 우선순위 적용), 각 마커 핀 클릭 시 카카오 로드뷰 및 카카오맵 연동 링크 제공), 우측 탭에 분석 실행 버튼 클릭 시 기존 FactoryON 실거래가 대신 Neon DB에서 로드한 예측 생산성 데이터(dan_productivity)를 원 단위로 변환하여 천㎡당 예측 생산성 및 모델 출처 카드로 제공합니다. AI 에이전트의 세부 자격 판정 보고서(매칭 업종 및 근거 요약)도 함께 상세 정보 탭에 제공됩니다.
